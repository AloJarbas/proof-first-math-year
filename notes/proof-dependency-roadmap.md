# Proof dependency roadmap

This map is for one practical question:

**what should unlock what, and what should stay light until the base is real?**

![Proof dependency roadmap](../assets/proof-dependency-roadmap.svg)

## The main claim

A lot of self-study trouble comes from treating every subject as if it should start at full intensity on day one.

This packet argues for a different order:

1. make proof-writing normal,
2. bring calculus back as the main computational spine,
3. use discrete math as reinforcement instead of overload,
4. study linear algebra more seriously once the language and notation stop fighting you,
5. let real analysis become the maturity core on prepared ground,
6. do probability after that base is strong enough to carry it cleanly.

## How to use the map

The arrows are not all the same kind.

- **Blue arrows** show the main knowledge flow.
- **Orange arrows** show load-management advice.

That means some links are about prerequisites and some are about pacing.
The pacing links matter just as much, because poor sequencing is often a workload problem before it becomes a content problem.

## The most important node

Proof foundations are first because they remove friction everywhere else.
If definitions, quantifiers, contradiction, and induction still feel foreign, every later subject gets more expensive than it should be.

That is why the proof journal, error tags, and weekly review ritual matter so much here.
They are not side paperwork. They are the mechanism that turns reading into control.

## The subtle point

Linear algebra is important enough that it should appear early, but not so early that it turns into one more language the learner cannot yet speak.
That is why the repo keeps a distinction between:

- a **light preview** when proof work is still settling,
- a **fuller pass** once the notation and structure start to feel natural.

## What this map is trying to prevent

- four equal-intensity courses started at once
- calculus swallowing the whole year again
- proof practice collapsing into passive reading
- linear algebra starting so hard that everything fragments
- probability arriving before the learner can handle careful hypotheses comfortably

## Best use

Use this map during weekly review.
If the current study plan does not match the dependency flow, ask two questions:

1. which stream is actually primary right now?
2. which stream should be demoted back to preview status until the base is stronger?
